## REVIEW-01
- Source doc: conversation-approved-plan
- Review agent: same-model sub-agent
- Scope checked: project-only CLI installation, test isolation, 9 workflows, 44 curated skills, 23 core plus stack agents, balanced hooks and memory, dual plugin manifests, marketplace metadata, Codex command/event compatibility, update/uninstall protection, and validation evidence.
- Evidence checked: current diff and CSV, isolated `npm run harness:test`, isolated `npm test` (`3087/3087`), Claude strict validator, Codex Python plugin validator, 18 workflow quick validators, npm pack dry-run, ESLint, global registration audit.
- Claim/evidence alignment: mismatches found
- Limited validation honestly reported: yes
- Result: gaps_found
- Gaps:
  - The CLI accepted arbitrary non-Git directories and did not resolve nested invocations to a Git worktree root.
  - The test isolation guard used lexical paths without realpath verification, allowing a project symlink or junction to resolve outside the sandbox.
  - `memory.maxInjectedCharacters` was validated in config but ignored by the hook runtime; SessionStart used a hard-coded 8000-byte source read and could emit more than the configured cap.
  - Stack detection coverage missed pure C, Xcode/Swift directory names, `.sln`/`global.json`, and Kubernetes YAML manifests.
  - Hook runtime data paths trusted payload cwd and symlinked `.aster`/plugin data roots without a realpath containment check.
  - Conditional project-local MCP health checks were not wired into the selected hook surface.
  - Real marketplace/plugin installation, skill visibility, new-task reload, and `/hooks` trust were intentionally not run because the user required project-only tests; static validators cannot prove those runtime claims.
- Follow-up issues added: HARNESS-06, HARNESS-07, HARNESS-08, HARNESS-09, HARNESS-10, HARNESS-11, REVIEW-02
- Assumptions: The user's project-only test constraint takes precedence over the plan's optional isolated marketplace smoke test.
- Decision debt: The dual-platform plugin keeps a Codex-compatible `Stop` hook; project-copy Claude installs use `SessionEnd`. Runtime semantics for Claude plugin-only installs remain a limited validation item.
- Human-required blockers: none for the code fixes; real plugin smoke testing remains user-authorized only.
